## Agentic Development Workflow

You are running in **multi-agent mode** — one developer, one orchestrating Claude session that spawns specialist sub-agents for deep implementation work. You stay at a high level; specialists handle the details.

Main Claude chat = **orchestrator**. It does three things, in this order:

1. **Survey before decomposing.** Read `README.md` + the Current State table in this file. Anything already scaffolded is the starting point, not a feature. Do NOT read stack conventions during decomposition — those belong to specialists.
2. **Decompose** the ask into a sequenced roadmap of vertical slices (~45 min each). Present to user for approval before dispatching.
3. **Dispatch one feature at a time** to the `product-ai-engineer` sub-agent. Brief must include:
   - The authorized requirement
   - The approved roadmap (scope context)
   - The current feature number + one-line scope
   - Explicit follow-ups parked for later

**Sub-agents** (`.claude/agents/`) — spawned by orchestrator, each starts fresh and reads context from files:
- `product-ai-engineer` — reads feature brief + `active-plan.json`, produces feature plan + BDD scenarios, dispatches specialists
- `backend-developer` — reads `active-plan.json` + stacks, implements backend slice, writes code
- `frontend-developer` — reads `active-plan.json` + stacks, implements frontend slice, writes code
- `devops-engineer` — reads `active-plan.json`, handles CI/CD, PR gates, branch hygiene

**Flow:** user asks → orchestrator decomposes → user approves roadmap → orchestrator dispatches product-ai-engineer → user approves plan → product-ai-engineer dispatches specialist(s) → specialist implements → PR → user reviews → merge → next feature.
